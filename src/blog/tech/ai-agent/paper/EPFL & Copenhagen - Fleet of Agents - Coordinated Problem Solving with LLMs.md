*论文发布时间：2024-05-10*

EPFL 和哥本哈根大学联合发布的 **Fleet of Agents（FoA）**（论文标题为《[Fleet of Agents: Coordinated Problem Solving with Large Language Models](https://arxiv.org/abs/2405.06691)》）提出了一种基于**遗传粒子滤波（Genetic Particle Filtering）** 的 LLM 推理框架——派出一支 Agent 舰队并行探索搜索空间，然后通过重采样机制把资源集中到最有希望的路径上。
核心成果：**在 Game of 24、Mini Crosswords、WebShop 三个基准上，FoA 平均质量提升 ~5%，同时只花了此前 SOTA 方法 ~40% 的成本**。

![Figure 1: Game of 24 任务上各方法的成本-质量权衡分析（GPT-3.5）。FoA 在所有可能的价格点上都显著优于现有 SOTA 方法，实现了最佳的成本-质量平衡。](https://arxiv.org/html/2405.06691v3/x1.png)

## FoA 解决了什么问题？

**核心问题：现有的多步推理框架要么质量不够，要么成本太高——缺乏有效的成本-质量权衡机制。**

2024 年，LLM 推理框架已经形成两大阵营：

```
单次查询方法（Single-query）：
──────────────────────────────
  CoT, CoT-SC, Algorithm of Thoughts
  ✓ 便宜（只调用一次或少数次 LLM）
  ✗ 质量有限（一条链推理，容易走偏）
  ✗ 不适合需要环境交互的决策任务

多次查询方法（Multi-query）：
──────────────────────────────
  ToT, GoT, LATS, RAP
  ✓ 质量好（搜索多条路径，择优选择）
  ✗ 成本爆炸（搜索树可能指数级膨胀）
  ✗ 无法精确控制宽度和深度 → 延迟和成本不可预测

  具体例子（WebShop 任务）：
    LATS: 平均分 66.1，花费 $232
    Act:  平均分 58.1，花费 $0.10
    → 为了 8 分的提升，花了 2300 倍的钱
```

**根本矛盾：** Tree-of-Thoughts 类方法通过扩展"最有希望的 c 个状态"来搜索，但每一层扩展多少节点是不可控的——可能在某一层爆炸式膨胀。FoA 的解决思路是：**固定 Agent 数量（树宽度）和步数（树深度），通过重采样来动态调整探索方向**。

## 这个问题真实存在吗？

**非常真实，且是 LLM Agent 大规模部署的核心瓶颈。**

*   **LLM 推理成本占生命周期的大头：** 训练成本是一次性的，但推理成本随用户量线性增长。如果每个推理任务都要花 $200+（如 LATS 在 WebShop 上），商业化根本不可行。
*   **强模型不等于免费：** GPT-4 的 API 价格是 GPT-3.5 的 15-60 倍。即便用开源模型，推理资源也是有限的。
*   **现有方法缺乏成本意识：** 在 FoA 之前，几乎没有论文**同时报告质量和成本**。大家只比准确率，不比花了多少钱。

## 为什么现在才有人去解决？

1.  **推理框架的繁荣：** 2023-2024 年涌现了 CoT、ToT、GoT、LATS、Reflexion、RAFA 等大量框架，但它们都在追求更高的准确率，没有人系统性地关注成本-质量权衡。FoA 是第一个把这个问题提到台面上的工作。
2.  **跨领域理论迁移成熟：** FoA 的核心算法"粒子滤波"来自统计学和信号处理领域（下文会详细解释这是什么），已在机器人定位、车辆路径规划等场景验证了几十年。但在 LLM 出现之前，没有"需要多个智能体并行搜索解空间"的应用场景。
3.  **开源模型性能的提升：** FoA 的一个重要发现是 LLaMA3.2-11B + FoA 能超越裸跑的 LLaMA3.2-90B——这意味着**框架可以弥补模型规模的差距**，这在小模型不够强的时代是做不到的。

## 它是如何解决的？

### 先搞懂"粒子滤波"到底是什么

在看 FoA 的具体设计之前，必须先理解它借鉴的核心思想。不要被"遗传粒子滤波"这个名字吓到——它的直觉非常简单：

**想象你蒙着眼在一个迷宫里找出口。** 你只有一个人，只能一步一步摸索，效率很低。

粒子滤波的思路是：**不要派一个人，派 100 个人（"粒子"）同时从起点出发，各自随机摸索。** 每走几步，暂停一下，看看谁离出口最近。然后把走得最远的人"克隆"几份，把走进死胡同的人"传送"到走得好的人的位置上。然后继续走。

```
粒子滤波的直觉（迷宫类比）：
─────────────────────────────

第 0 步: 5 个人从起点出发
  🧑A ──→ 往北走
  🧑B ──→ 往东走
  🧑C ──→ 往南走（死胡同）
  🧑D ──→ 往东走
  🧑E ──→ 往北走

第 1 步: 暂停，评估每个人离出口多远
  A: 离出口还有 8 步  ← 还行
  B: 离出口还有 3 步  ← 很好！
  C: 死胡同            ← 淘汰
  D: 离出口还有 6 步  ← 一般
  E: 离出口还有 7 步  ← 一般

重采样: 按"离出口近"的概率，有放回地抽 5 个人
  → B 被选中 3 次（克隆 3 份），D 被选中 1 次，E 被选中 1 次
  → A 和 C 被淘汰
  → 现在 3 个人站在 B 的位置，1 个在 D 的位置，1 个在 E 的位置

第 2 步: 5 个人从各自的新位置继续随机探索...
  → 3 个人从 B 的位置出发，但各自走不同方向（保持了一定多样性）
  → 资源自动集中到了最有希望的方向
```

这就是粒子滤波的全部核心：**多个粒子并行探索 → 评估谁做得好 → 把资源集中到好的方向 → 继续探索。**

"遗传"则是借鉴了生物进化的术语：每个粒子独立探索叫"变异"（mutation），按好坏筛选叫"选择"（selection）——本质就是达尔文的"物竞天择、适者生存"。

### FoA 如何把粒子滤波用到 LLM 推理上

FoA 的做法就是：**把每个 LLM Agent 当作一个"粒子"**。每个 Agent 独立调用 LLM 来推进自己的求解路径，然后定期暂停，用 LLM 评估每条路径的好坏，把 Agent 重新分配到最有希望的路径上。

![Figure 2: FoA 与传统树搜索方法的对比。ToT/GoT/LATS 每步扩展 c 个最有希望的状态，搜索树可能指数级膨胀；FoA 固定 n 个 Agent 和 t 步深度，通过重采样精确控制宽度和深度，成本可预测。](https://arxiv.org/html/2405.06691v3/x2.png)

![Figure 3: FoA 运行示意。n=5 个 Agent 各自独立探索 k 步（变异阶段），然后根据价值函数重采样（选择阶段），将资源集中到高价值状态上。](https://arxiv.org/html/2405.06691v3/x3.png)

### 用 Game of 24 走一遍完整流程

Game of 24 的规则：给定 4 个数字，用 +-×÷ 组合得到 24。比如给定 `4 9 10 13`，答案是 `4 × (9 - (13 - 10)) = 24`。

假设 FoA 配置：**3 个 Agent，每轮独立探索 1 步（k=1），共走 3 步（因为 3 步恰好用完 4 个数）**。

```
═══════════════════════════════════════════════════════════════
  题目: 用 4, 9, 10, 13 算 24
═══════════════════════════════════════════════════════════════

━━━ 第 1 轮: 变异（每个 Agent 各自尝试一步运算）━━━

  Agent A: "13 - 10 = 3"   → 剩余 [3, 4, 9]
  Agent B: "13 - 9 = 4"    → 剩余 [4, 4, 10]
  Agent C: "10 + 9 = 19"   → 剩余 [4, 13, 19]

━━━ 第 1 轮: 选择（让 LLM 评估每条路走下去能不能凑到 24）━━━

  LLM 评估 Agent A [3, 4, 9]:
    "3 × 4 = 12, 12 + 9 = 21... 不太行"
    "9 - 3 = 6, 6 × 4 = 24!  可以！"  → 评分: sure ✓

  LLM 评估 Agent B [4, 4, 10]:
    "4 × 4 = 16, 16 + 10 = 26... 不行"
    "10 + 4 + 4 = 18... 不行"
    "10 × 4 - 4 = 36... 不行"  → 评分: impossible ✗

  LLM 评估 Agent C [4, 13, 19]:
    "19 + 4 = 23... 差一点"
    "19 - 13 = 6, 6 × 4 = 24!  可以！"  → 评分: sure ✓

  重采样（按评分加权，有放回抽 3 个）:
    A 的评分高 → 被选中 2 次（克隆一份）
    C 的评分高 → 被选中 1 次
    B 的评分低 → 被淘汰

    重采样后:
    Agent A₁: [3, 4, 9]    ← 原 A 的状态
    Agent A₂: [3, 4, 9]    ← A 的克隆
    Agent C:  [4, 13, 19]   ← 原 C 的状态

━━━ 第 2 轮: 变异（各自再走一步）━━━

  Agent A₁: "9 - 3 = 6"  → 剩余 [4, 6]
  Agent A₂: "4 × 3 = 12" → 剩余 [9, 12]
  Agent C:  "19 - 13 = 6" → 剩余 [4, 6]

  （注意: A₁ 和 A₂ 虽然从同一状态出发，但各自随机选了不同运算
    → 这就是"克隆后保持多样性"的关键）

━━━ 第 2 轮: 选择（评估）━━━

  A₁ [4, 6]: "4 × 6 = 24!"  → sure ✓
  A₂ [9, 12]: "9 + 12 = 21... 12 - 9 = 3..."  → impossible ✗
  C  [4, 6]: "4 × 6 = 24!"  → sure ✓

  重采样: A₁ 和 C 被保留，A₂ 被淘汰

━━━ 第 3 轮: 最后一步 ━━━

  Agent A₁: "4 × 6 = 24"  → ✅ 找到解！
  → 回溯路径: 13-10=3, 9-3=6, 4×6=24
  → 即 4 × (9 - (13-10)) = 24
```

**对比一下如果只有 1 个 Agent（类似 CoT）：** 如果它第一步选了 `13 - 9 = 4`（Agent B 的路），后续无论怎么算 `[4, 4, 10]` 都凑不出 24，就直接失败了。而 FoA 通过**3 个 Agent 并行试不同的第一步**，再**淘汰走不通的、增援走得通的**，大幅提高了成功率。

### 三个核心机制详解

理解了上面的例子，再看 FoA 的三个机制就很清楚了：

**机制一：变异阶段（每个 Agent 独立走 k 步）**

```
每个 Agent 独立调用 LLM 决定下一步行动:
  • Game of 24: 选两个数做一次运算
  • WebShop: 执行一个搜索/点击动作
  • Crosswords: 填一个单词

两个额外规则:
  • 强制变异: Agent 不能重复上一步（避免原地踏步）
  • 突然死亡: 如果 Agent 走进明确的死胡同（如 Game of 24 中
    所有数字组合都不可能凑到 24），立即将它传送到其他 Agent
    的位置上，不浪费名额
```

**机制二：选择阶段（评估 + 重采样）**

这是 FoA 的核心。每 k 步暂停一次，做两件事：

```
① 评估: 用 LLM 给每个 Agent 的当前状态打分
   → 和 ToT 一样，让 LLM 判断 "sure / likely / impossible"

② 重采样: 按打分概率，有放回地重新抽 n 个 Agent
   → 打分高的状态可能被多个 Agent "占据"（克隆）
   → 打分低的状态上的 Agent 被淘汰
   → 打分都差不多 → 大家都保留（维持多样性）

   三种抽样策略:
   • 线性加权:  分越高被抽中概率越大，但低分也有机会
   • 指数加权:  更激进地向高分集中
   • 贪心:     只保留最高分（完全不探索，通常效果最差）
```

**为什么重采样比 ToT 的"选 Top-c 扩展"更好？** ToT 每一步都固定扩展最好的 c 个节点，但 c 是写死的——有时候只有 1 个好方向，c=5 就浪费了；有时候有 4 个好方向，c=2 就遗漏了。FoA 的重采样是**自适应的**：好方向多就都保留，好方向少就集中兵力。

**机制三：回溯（从历史好状态重新出发）**

```
普通重采样: 只从当前 n 个 Agent 的状态中抽
回溯重采样: 还可以从之前某一轮的状态中抽

  比如第 3 轮时所有 Agent 都走进了死胡同，
  但第 1 轮时 Agent B 的状态其实不错
  → 回溯机制允许把某个 Agent "传送"回第 1 轮 B 的位置，重新来过

  折扣因子 γ 控制"回溯多远":
    γ = 0:   不允许回溯
    γ = 0.5: 越早的状态被选中概率越低（鼓励向前）
    γ = 1:   所有历史状态一视同仁（可能导致反复回溯、浪费步数）
```

### 为什么成本可控？

```
FoA 的 LLM 调用次数是精确可算的:

  变异阶段: 每轮每个 Agent 调用 1 次 LLM → n 次
  选择阶段: 每轮对 n 个状态各调用 1 次 LLM → n 次
  总共 t 步，每 k 步选择一次:
    总调用 = n × t (变异) + n × (t/k) (选择)

  比如 Game of 24: n=9, t=9, k=1
    → 9×9 + 9×9 = 162 次 LLM 调用，固定的！

  而 ToT: 每步扩展 c=5 个节点，3 层深
    → 最多 5³ = 125 个叶节点，但实际还要加上评估调用
    → 总调用次数取决于搜索树的实际形状，不可预测
```

**FoA 不是一种 prompting 方案，而是一个 runtime**——它可以搭配任何现有的 Agent 或 prompting 策略使用，不需要额外的 prompt 设计。论文实验直接复用了 ToT 和 LATS 的 prompt，没有做任何定制。

### 消融实验

![Figure 6: 消融实验——移除选择阶段、重采样、回溯、缓存、批处理的影响。每个组件都对性能有正向贡献。](https://arxiv.org/html/2405.06691v3/x6.png)

关键发现：

*   **选择阶段不可或缺：** 去掉后成功率暴跌，因为没有选择 = Agent 各自为战、不共享信息。
*   **重采样策略有效：** 改成"全选最高价值"会降低多样性，略微降低质量。
*   **回溯是安全网：** γ=0（无回溯）时，所有 Agent 走进死胡同就无法恢复；γ=1（无折扣）则历史状态太多导致噪声。γ=0.5 是最佳平衡。

## 还有更好的解决方案吗？

FoA 的框架有明确的改进方向：

*   **固定舰队规模的局限：** 目前所有任务用相同的 Agent 数量。更理想的做法是根据任务难度动态调整——简单任务少派几个 Agent，难的任务多派。
*   **同质 Agent 的限制：** 目前所有 Agent 使用相同的策略。如果不同 Agent 用不同的 prompt 或不同的搜索策略（异质舰队），可能进一步提升多样性。
*   **价值函数的质量：** FoA 的重采样完全依赖 LLM 作为价值函数的准确性。如果评估不靠谱（和 [ToT](./Princeton%20%26%20Google%20-%20Tree%20of%20Thoughts%20-%20Deliberate%20Problem%20Solving%20with%20LLMs.md) 面临同样的问题），重采样就会误导方向。
*   **与 Reflexion 的互补：** FoA 做的是**同一 trial 内的并行搜索**，[Reflexion](./Princeton%20%26%20Northeastern%20-%20Reflexion%20-%20Language%20Agents%20with%20Verbal%20Reinforcement%20Learning.md) 做的是**跨 trial 的经验积累**。两者结合——"多 Agent 并行探索 + 失败后反思改进"——可能是更强的方案。

## 冷思考：从搜索算法到 Agent 运行时

### 1. FoA 的真正身份：不是推理框架，是 Agent 的"操作系统"

FoA 和 ToT/GoT/Reflexion 有一个本质区别——它不规定 Agent 怎么思考，只规定 Agent 群体怎么协调。任何现有的 Agent（ReAct Agent、CoT Agent、甚至 VOYAGER 式的代码 Agent）都可以作为 FoA 的"粒子"。

```
FoA 的定位：
────────────

  ToT/GoT:    "一个 Agent 怎么搜索更好的思路"  → 搜索算法
  Reflexion:  "一个 Agent 怎么从失败中学习"     → 学习算法
  VOYAGER:    "一个 Agent 怎么积累技能"          → 终身学习
  FoA:        "一群 Agent 怎么协调探索"          → 协调运行时

  → FoA 是正交于上述所有方法的
  → FoA + ToT = 舰队中每个 Agent 自己做树搜索
  → FoA + Reflexion = 舰队中每个 Agent 自己会反思
```

### 2. o1/R1 内化了 ToT，为什么没人内化 FoA？

OpenAI o1 和 DeepSeek-R1 通过 RL 训练，让模型学会了在一条长链里自己"探索-评估-回溯"——本质上是把 ToT 的搜索能力**内化到了模型权重里**。那自然的追问是：能不能也把 FoA 的并行协调策略内化？

**答案是不能，因为 FoA 的核心优势来自并行，而并行无法被单个模型内化。**

ToT 能被内化，是因为树搜索本质上是**顺序的**——"试一条路 → 不行退回来 → 换一条路"，可以被压成一条文本链。模型学会写"等等，这不对，换个思路..."就完成了内化。

但 FoA 的核心是 **N 个 Agent 同时走 N 条不同的路，然后比较谁走得好**。一个 Transformer 同一时刻只生成一个 token，没法真的"同时走 5 条路"。如果把并行变成串行（先走完路 A，再走路 B...），那就退化成了"跑 N 次 CoT 最后投票"——就是 CoT-SC，丢掉了 FoA "中途动态调兵"的核心优势。

```
可内化 vs 不可内化的分界线：
────────────────────────────

可内化（单流程内可完成）:
  ToT 的搜索回溯    → o1/R1 学会了 "wait, let me reconsider..."
  Reflexion 的反思   → o1/R1 学会了 "之前的方法有问题，因为..."

不可内化（需要真正的并行）:
  FoA 的多路径协调  → 需要多个模型实例同时运行 + 外部调度
  CoT-SC 的多路采样 → 需要多次独立调用 + 外部投票
```

**这恰恰是 FoA 的长期价值：** o1/R1 再强，也只是一个人想得更深；FoA 是派一群人同时想不同方向。两者完全互补——FoA + o1 = 一群会深度思考的 Agent 在协调探索，理论上比单个 o1 更强（当然也更贵）。

### 3. CoT-SC 也被内化了吗？外部多路径还有必要吗？

顺着上面的逻辑自然会问：既然 ToT 被内化了，CoT-SC 是不是也被内化了？那还需要外部跑多条路径吗？

**答案：部分内化了，但核心优势无法内化。**

CoT-SC 有两个组成部分：① "尝试多种思路"——这已经被 o1/R1 内化了，模型会写"换个方法试试..."；② "N 条独立链 + 投票"——这无法内化，因为**统计独立性**是关键。

```
内化的多路径 vs 外部的多路径：
──────────────────────────────

内化（o1/R1 的长链推理）:
  一个人先想方案 A，觉得不对，再想方案 B
  → B 的思路已经被 A 污染了（前面的 token 在 context 里）
  → 不是真正的统计独立
  → 类似"一个人换了个角度再想想"

外部 CoT-SC:
  5 个独立的模型调用，各自生成答案，最后投票
  → 每次调用互不影响，错误互不相关
  → 投票能有效消除随机错误
  → 类似"找 5 个人各自独立思考"

区别的本质:
  统计独立性 → 错误不相关 → 投票/重采样才有效
  单次生成中的"多次尝试" → 错误相关 → 多样性打折扣
```

**所以外部多路径的必要性取决于任务难度：**

*   **简单任务**：o1/R1 内化的推理足够，不需要外部多路径。
*   **中等任务**：外部多路径仍有价值，但 FoA 比 CoT-SC 更值得用——同样花钱跑多个实例，FoA 的动态重采样比 CoT-SC 的盲投票更聪明。
*   **前沿任务**（接近模型能力极限）：外部多路径几乎必须，真正的统计独立性不可替代。

事实上，o1 的 API 背后很可能仍在做外部多路径采样（best-of-N），只是用户看不到。**CoT-SC 的必要性没有消失，只是使用场景收窄到了模型能力边界附近——而 FoA 是这个场景下比 CoT-SC 更好的选择。**

### 5. 进化算法的理论后盾

FoA 的"变异-选择"循环和达尔文进化论是同构的。这不只是修辞——进化算法（Evolutionary Algorithms）在优化领域已研究了 50+ 年，有大量可借鉴的理论成果：多样性维护（避免所有 Agent 收敛到同一个烂方向）、精英策略（始终保留历史最优解）、自适应变异率（初期大幅探索、后期微调）。FoA 目前只用了最基础的重采样，还有很大的改进空间。

### 6. 在 Agent 族谱中的定位

```
2023-2024 Agent 推理范式图谱（按"多路径如何协调"分类）：

  单路径:
    IO → CoT                    一条链走到底，不回头

  多路径 / 无协调:
    CoT-SC                      采样 N 条链，各走各的，最后投票选答案
                                路径之间完全不交互

  多路径 / 静态搜索:
    ToT → GoT → LATS → RAP     展开搜索树/图，按固定策略选分支扩展
                                分支数 c 是预设的，不会根据中间结果调整

  多路径 / 动态协调:             ← FoA 的位置
    FoA                         N 个 Agent 并行探索，定期评估并重新分配兵力
                                兵力分配根据中间结果自适应调整

  跨轮学习:
    Reflexion → RAFA            单次 trial 内可以用上述任何方式
                                关键创新在跨 trial 的经验积累

  终身学习:
    VOYAGER, Generative Agents  不断积累技能/记忆，跨任务复用
```

FoA 的真正对手不是 Reflexion 或 VOYAGER（它们解决的是不同维度的问题），而是 **CoT-SC 和 ToT**——三者都是"多条路径怎么协调"的不同答案：CoT-SC 说"不协调，最后投票"；ToT 说"按固定规则选分支"；FoA 说"根据实际表现动态调兵遣将"。

## 总结

FoA 的核心思路一句话就能说清：**派一群 Agent 同时探索，定期淘汰走不通的、增援走得好的**——本质上就是把"物竞天择"用在了 LLM 推理上。

它的三个核心机制——**变异**（Agent 各走各的）、**选择**（评估打分、重新分配兵力）、**回溯**（走进死胡同可以退回去重来）——在三个基准任务上，用 SOTA 方法 40% 的成本获得了 5% 的质量提升；在 WebShop 上更是用 LATS 0.7% 的成本超出了 10 分。

这篇论文最深远的意义不在于具体数字，而在于它确立了一个新视角：**Agent 推理的改进不一定要发生在单个 Agent 内部（更好的 prompt、更深的搜索），也可以发生在 Agent 群体层面（更好的协调、更聪明的资源分配）**。
